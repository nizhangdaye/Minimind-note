# 一、为什么需要位置编码

- 在 RNN 中，模型按时间步顺序处理 Token，位置信息被隐式地编码在隐状态中。但 Transformer 通过注意力机制**一次性并行处理**序列中的所有 Token，打破了线性顺序，因此急需一种新方法来注入位置信息。
- 在注意力机制中，Q 和 K 的点积操作本身是**位置无关**的——"人咬狗"和"狗咬人"在并行 Attention 视角下是完全相同的 Token 集合。如果不加干预，模型无法区分谁是施暴者、谁是受害者。

# 二、绝对位置编码（APE）

位置编码的发展史，本质上是人类用数学语言描述"序列秩序"的探索史。绝对位置编码（Absolute Positional Encoding）是这一探索的起点，其核心思路是：**为句子中的每一个固定位置生成一个专属的"坐标向量"，并将其直接与词的语义向量（Embedding）相加。**

根据生成方式的不同，分为两大流派：

## 2.1 正弦位置编码（Sinusoidal PE）

这是 Transformer 开山之作《Attention Is All You Need》中提出的经典方案。它不依赖模型训练，而是通过固定的数学公式直接计算得出。

**核心公式：**

$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$

$$PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$

> [!info] 变量说明
> - $pos$：Token 在序列中的绝对位置（0, 1, 2, ...）
> - $i$：维度索引（$i = 0, 1, \dots, d_{model}/2 - 1$）
> - $d_{model}$：词向量的总维度
> - $2i$ 和 $2i+1$：偶数维度用 sin，奇数维度用 cos

**多尺度时钟隐喻：** 这种编码可以被视作一组转速不同的时钟
- **低维分量（小 $i$，高频）：** 像转速飞快的"秒针"，波长短，区分紧密相邻的位置。
- **高维分量（大 $i$，低频）：** 像缓慢移动的"时针"，波长长，在宏观距离上进行定位。

**加法注入：** 位置向量直接与 Token Embedding 相加。虽然看似混合了两种毫不相干的信息，但由于高维空间的稀疏性，语义特征和位置特征往往分布在近似正交的子空间中，模型依然能够分别提取。

## 2.2 可学习的绝对位置编码（Learned APE）

GPT-2 和 BERT 等早期经典模型选择让模型自己"学"出位置感。

- **核心机制：** 初始化一个大小为 $L_{max} \times D$ 的参数矩阵（$L_{max}$ 是支持的最大长度），每个位置有一个专属向量，随模型训练一起更新。
- **优势：** 灵活性极强，能根据数据分布自适应地学习位置表示。
- **致命缺陷：彻底丧失外推能力。** 训练时最多见过 1024 个词，推理时输入 1025 就会因找不到位置向量而崩溃。这成为早期大语言模型扩展上下文窗口的死穴。

# 三、旋转位置编码（RoPE）

绝对位置编码虽然解决了从无到有的"秩序"问题，但自然语言的本质在于词与词之间的**相对距离**（如"虽然...但是..."之间的关联跨度），而非绝对排在第几。加上"无法外推"的硬伤，促使学术界寻找更优雅的方案。

RoPE（Rotary Positional Embedding）彻底改变了格局——摒弃了传统的"加法（APE）"和"偏置（ALiBi）"，转而采用**复数空间的乘法（旋转）**。

## 3.1 二维直觉：在复平面上"转圈"

为了理解 RoPE，先看最简单的二维情形。将 Embedding 向量的每两个维度视为复平面上的一个复数。

设位置 $m$ 处的 Query 向量 $\boldsymbol{q}$ 和位置 $n$ 处的 Key 向量 $\boldsymbol{k}$ 的原始复数表示为：

$$\boldsymbol{q} = r_q e^{i\theta_q}, \quad \boldsymbol{k} = r_k e^{i\theta_k}$$

> [!info] 变量说明
> - $r_q, r_k$：向量的模长（幅值），反映语义强度
> - $\theta_q, \theta_k$：向量的初始辐角（相位），反映语义方向
> - $e^{i\alpha} = \cos\alpha + i\sin\alpha$：欧拉公式，表示复平面上的单位旋转

**RoPE 的核心操作**是为每个位置乘上一个旋转因子：

$$f(\boldsymbol{q}, m) = \boldsymbol{q} \cdot e^{im\theta} = r_q e^{i(\theta_q + m\theta)}$$

$$f(\boldsymbol{k}, n) = \boldsymbol{k} \cdot e^{in\theta} = r_k e^{i(\theta_k + n\theta)}$$

> [!info] 变量说明
> - $m, n$：Token 的绝对位置索引
> - $\theta$：该维度对的旋转频率（角速度）
> - $e^{im\theta}$：位置 $m$ 的旋转因子，几何上相当于把向量逆时针旋转 $m\theta$ 角度

注意：此时向量上编码的仍然是**绝对位置**。

## 3.2 见证奇迹：内积中浮现相对位置

Transformer 的 Attention 本质上是计算 Q 和 K 的内积。在复数域中，内积对应于共轭乘积的实部：

$$\langle f(\boldsymbol{q}, m), f(\boldsymbol{k}, n) \rangle = \text{Re}\left[ f(\boldsymbol{q}, m) \cdot f(\boldsymbol{k}, n)^* \right]$$

代入旋转后的公式展开：

$$= \text{Re}\left[ \boldsymbol{q} e^{im\theta} \cdot \boldsymbol{k}^* e^{-in\theta} \right] = \text{Re}\left[ \boldsymbol{q} \boldsymbol{k}^* \cdot e^{i(m-n)\theta} \right]$$

> [!tip] 核心结论：平移不变性
> 绝对位置 $m$ 和 $n$ 消失了，留下的只有**相对距离 $(m-n)$ **。模型虽然对每个词做的是绝对位置的旋转，但计算注意力分数时，完全只取决于它们之间的相对距离。

## 3.3 推广到高维：分块旋转矩阵

在实际模型中，词向量维度 $d$ 远不止二维（如 4096 维）。RoPE 的做法是**分而治之**：将 $d$ 维向量两两分组，切分成 $d/2$ 个二维子空间，每个子空间独立旋转。

**旋转矩阵（分块对角形式）：**

$$\mathbf{R}_{\Theta, m} = \begin{pmatrix} \cos m\theta_0 & -\sin m\theta_0 & & \\ \sin m\theta_0 & \cos m\theta_0 & & \\ & & \cos m\theta_1 & -\sin m\theta_1 \\ & & \sin m\theta_1 & \cos m\theta_1 \\ & & & & \ddots \end{pmatrix}$$

**每个子空间的旋转频率 $\theta_j$ 遵循几何级数递减：**

$$\boxed{\theta_j = \text{base}^{-2j/d}}$$

> [!info] 变量说明（对应代码 `步骤 1`）
> - $j$：维度对索引（$j = 0, 1, \dots, d/2 - 1$）
> - $d$：`head_dim`，每个注意力头的维度
> - $\text{base}$：基频参数（通常为 $10000$），即代码中的 `rope_base`
> - 低维度（小 $j$）：$\theta_j$ 大 → 旋转快（像秒针），区分紧邻位置
> - 高维度（大 $j$）：$\theta_j$ 小 → 旋转慢（像年针），保持长距离连贯

**对应代码：**
```python
freqs = 1.0 / (rope_base ** (torch.arange(0, dim, 2)[: (dim // 2)].float() / dim))
```

**位置 $m$ 处每个维度对的旋转角度为 $m \cdot \theta_j$：**

$$\Phi_{m,j} = m \cdot \theta_j$$

**对应代码：**
```python
t = torch.arange(end)         # 位置索引 [0, 1, ..., end-1]
freqs = torch.outer(t, freqs) # 外积 → [end, dim//2] 的角度矩阵
```

**最终对 Q/K 的旋转操作**等价于（对应代码 `apply_rotary_pos_emb`）：

$$\boxed{\text{RoPE}(\boldsymbol{x}, m) = \boldsymbol{x} \odot \cos(m\boldsymbol{\theta}) + \text{rotate\_half}(\boldsymbol{x}) \odot \sin(m\boldsymbol{\theta})}$$

> [!info] 变量说明
> - $\boldsymbol{x}$：Q 或 K 向量
> - $\odot$：逐元素乘法
> - $\text{rotate\_half}$：将向量后半部分取负放到前面，即 $[a,b,c,d] \to [-c,-d,a,b]$
> - 这是复数旋转 $\boldsymbol{x} \cdot e^{im\theta}$ 的实数等价形式

**对应代码：**
```python
q_embed = (q * cos) + (rotate_half(q) * sin)
k_embed = (k * cos) + (rotate_half(k) * sin)
```

## 3.4 数学特性带来的三大优势

1. **正交变换保模长：** 旋转矩阵 $\mathbf{R}_{\Theta, m}$ 是正交矩阵，不改变向量长度，**零语义污染**。
2. **长程自然衰减：** 随着相对距离 $|m-n|$ 增大，高频部分相位差趋于随机，内积期望自然趋近 0，符合"越远关联越弱"的直觉。
3. **KV Cache 兼容：** 旋转操作可直接注入推理时的 KV 缓存体系，效率极高。

# 四、长上下文外推：YaRN

虽然 RoPE 理论上支持无限长度，但实际工程中，当推理长度超越训练时的最大长度（如训练 4 k，推理 8 k），模型性能会断崖式下跌（PPL 爆炸）。本章梳理解决**外推性故障**的三部曲：直接外推为何崩溃 → 线性内插的局限 → NTK 感知缩放 → YaRN 集大成。

## 4.1 为什么直接外推会崩溃？

**根源是 OOD（分布外）问题。** 回到"多级时钟"隐喻：神经网络只记住了训练窗口内（如 $0 \sim 4000$）的旋转组合。一旦输入长度达到 4001，旋转角度就进入了模型从未见过的数值区域，导致 Attention Score 异常，Softmax 分布崩塌。

## 4.2 线性内插（PI）：简单但粗暴

Meta 提出了一种"时空压缩"方案：既然长文本的位置值没见过，就等比例缩小，塞进已知范围。假设将窗口扩展 $s$ 倍，直接将位置索引 $m$ 替换为 $m/s$：

$$f_{\text{PI}}(\boldsymbol{q}, m) = \boldsymbol{q} \cdot e^{i \frac{m}{s} \theta}$$

> [!info] 变量说明
> - $s$：扩展倍数（如从 4 k 扩展到 32 k，则 $s = 8$）

**分辨率危机：** 虽然解决了 PPL 爆炸，但相邻 Token 间的旋转角度差也被压缩了 $s$ 倍：

$$\Delta \theta_{\text{PI}} = \frac{\Delta \theta_{\text{original}}}{s}$$

这就像对图片施加了高斯模糊——高频分量（原本用来区分相邻词的维度）被严重模糊，模型变得"老眼昏花"，丢失了局部语法的分辨力。

## 4.3 NTK-Aware Scaling：频域视角的觉醒

为解决分辨率丢失，社区引入了**神经正切核（NTK）理论**，核心洞察是：

**频谱偏差：** 神经网络倾向于优先学习低频信号，难以学习高频剧烈变化的信号。因此不能"一刀切"地压缩所有维度，必须**差异化对待**：

| 维度 | 频率 | 训练时状态 | 策略 |
|------|------|-----------|------|
| 低维（小 $j$） | 高频 | 已转了无数圈，外推无压力 | **不插值**，保持高分辨率 |
| 高维（大 $j$） | 低频 | 可能连半圈都没转完，极易 OOD | **强插值**，压缩回已知范围 |

**数学实现：** 通过放大 RoPE 的基频 $\text{base}$，让缩放因子从低维到高维平滑过渡：

$$b' = b \cdot s^{\frac{d}{d-2}}$$

> [!info] 变量说明
> - $b$：原始基频（`rope_base`）
> - $b'$：放大后的基频
> - $s$：扩展倍数
> - 效果：低维度频率几乎不变（保分辨率），高维度频率被强力压缩（防 OOD）

## 4.4 YaRN：集大成的精细化手术

NTK-Aware 虽惊艳，但处理超长文本（100 k+）时仍有瑕疵。**YaRN（Yet another RoPE extensioN）** 结合了分段频率处理和熵漂移修正，成为目前最完善的 RoPE 扩展框架。

### 4.4.1 NTK-by-parts（分段 NTK）

YaRN 放弃了平滑过渡，转而用**斜坡函数** $\gamma(j)$ 将维度明确划分为三个频段：

| 频段 | 条件 | $\gamma$ 值 | 策略 |
|------|------|-------------|------|
| 高频段 | $j < \text{low}$ | $\gamma = 0$ | 纯外推，完全不插值 |
| 中频段 | $\text{low} \le j \le \text{high}$ | $0 < \gamma < 1$ | 按比例混合 |
| 低频段 | $j > \text{high}$ | $\gamma = 1$ | 完全线性插值 |

**频率分段边界的计算：**

$$\text{inv\_dim}(\beta) = \frac{d \cdot \ln\!\left(\frac{L_{\text{train}}}{\beta \cdot 2\pi}\right)}{2 \ln(\text{base})}$$

$$\text{low} = \lfloor \text{inv\_dim}(\beta_{\text{fast}}) \rfloor, \quad \text{high} = \lceil \text{inv\_dim}(\beta_{\text{slow}}) \rceil$$

> [!info] 变量说明
> - $\beta_{\text{fast}}$：高频边界参数（默认 32），对应代码中的 `beta_fast`
> - $\beta_{\text{slow}}$：低频边界参数（默认 1），对应代码中的 `beta_slow`
> - $L_{\text{train}}$：训练时的最大长度，对应代码中的 `orig_max`
> - $\text{inv\_dim}(\beta)$：给定波长阈值 $\beta$，反算出对应的维度索引

**对应代码：**
```python
inv_dim = lambda b: (dim * math.log(orig_max / (b * 2 * math.pi))) / (2 * math.log(rope_base))
low = max(math.floor(inv_dim(beta_fast)), 0)
high = min(math.ceil(inv_dim(beta_slow)), dim // 2 - 1)
```

**斜坡函数 $\gamma(j)$ 实现线性过渡：**

$$\gamma(j) = \text{clamp}\!\left(\frac{j - \text{low}}{\text{high} - \text{low}},\; 0,\; 1\right)$$

**对应代码：**
```python
ramp = torch.clamp((torch.arange(dim // 2).float() - low) / max(high - low, 0.001), 0, 1)
```

**最终的分段频率缩放公式：**

$$\boxed{\theta'_j = \theta_j \cdot \left[(1 - \gamma(j)) + \frac{\gamma(j)}{s}\right]}$$

> [!info] 公式解读
> - 当 $\gamma = 0$（高频段）：$\theta'_j = \theta_j$，频率不变，保持高分辨率
> - 当 $\gamma = 1$（低频段）：$\theta'_j = \theta_j / s$，等价于线性内插，压缩回已知范围
> - 当 $0 < \gamma < 1$（中频段）：在两者之间线性混合

**对应代码：**
```python
freqs = freqs * (1 - ramp + ramp / factor)   # factor 即扩展倍数 s
```

### 4.4.2 温度缩放（Temperature Scaling）：修正熵漂移

这是 YaRN 最深刻的理论贡献。当序列变长，参与 Attention 计算的 Key 变多，Softmax 概率分布会偏离训练时的形态（**熵漂移**，变得更加平坦），破坏模型对关键信息的聚焦能力，导致"Lost in the Middle"长文失忆现象。

YaRN 引入温度系数 $t$ 修正 Attention logits：

$$\text{Attention}(Q, K) = \text{Softmax}\!\left(\frac{QK^T}{t \cdot \sqrt{d_k}}\right)$$

温度 $t$ 与扩展倍数 $s$ 的经验公式：

$$\boxed{\sqrt{\frac{1}{t}} = 0.1 \ln(s) + 1}$$

> [!info] 公式解读
> - 当 $s = 1$（不扩展）：$\sqrt{1/t} = 1$，即 $t = 1$，无变化
> - 当 $s$ 增大：$t < 1$，分母变小 → logits 变大 → Softmax 更尖锐，抵消了 Key 增多导致的注意力分散

**对应代码中的 `attn_factor`：**
```python
attn_factor = rope_scaling.get("attention_factor", 1.0)
# 最终 cos/sin 乘以 attn_factor，等效于缩放 Attention logits
freqs_cos = torch.cat([freqs.cos(), freqs.cos()], dim=-1) * attn_factor
freqs_sin = torch.cat([freqs.sin(), freqs.sin()], dim=-1) * attn_factor
```

借助这一修正，Llama 2 等模型仅需极少微调，就能完美扩展到 128 k 上下文。

# 五、公式 → 代码映射总览

> [!abstract] 全流程公式链
> **基础 RoPE：**
> 
> 1. 计算频率向量：$\theta_j = \text{base}^{-2j/d}$
> 2. 计算角度矩阵：$\Phi_{m,j} = m \cdot \theta_j$
> 3. 旋转 Q/K：$\text{RoPE}(\boldsymbol{x}, m) = \boldsymbol{x} \odot \cos(m\boldsymbol{\theta}) + \text{rotate\_half}(\boldsymbol{x}) \odot \sin(m\boldsymbol{\theta})$
>
> **YaRN 扩展（叠加在 RoPE 步骤 1 之后、步骤 2 之前）：**
> 
> 4. 计算分段边界：$\text{low}, \text{high} \leftarrow \text{inv\_dim}(\beta_{\text{fast}}), \text{inv\_dim}(\beta_{\text{slow}})$
> 5. 斜坡函数：$\gamma(j) = \text{clamp}\!\left(\frac{j - \text{low}}{\text{high} - \text{low}}, 0, 1\right)$
> 6. 频率缩放：$\theta'_j = \theta_j \cdot [(1 - \gamma(j)) + \gamma(j)/s]$
> 7. 温度修正：$\cos / \sin$ 乘以 `attn_factor`

# 六、完整实现代码

```python
def precompute_freqs_cis(
    dim: int,
    end: int = (32 * 1024),
    rope_base: float = 1e6,
    rope_scaling: Optional[dict] = None,
):
    """
    预计算 RoPE 的频率矩阵（cos 和 sin 值）。

    数学流程：
        1. θ_j = base^(-2j/d)           → 基础频率向量
        2. [YaRN] θ'_j = θ_j * ((1-γ) + γ/s)  → 分段频率缩放
        3. Φ_{m,j} = m * θ_j            → 角度矩阵（外积）
        4. 输出 cos(Φ), sin(Φ)           → 供 apply_rotary_pos_emb 使用

    Args:
        dim: 每个注意力头的维度（head_dim），即公式中的 d
        end: 最大序列长度（默认 32768）
        rope_base: 基频参数（默认 1e6），即公式中的 base
        rope_scaling: YaRN 外推配置字典，None 则不使用外推

    Returns:
        freqs_cos: [end, dim] 的 cos 值
        freqs_sin: [end, dim] 的 sin 值
    """
    # ===== 步骤 1：计算基础频率 =====
    # θ_j = base^(-2j/d)
    freqs, attn_factor = (
        1.0 / (rope_base ** (torch.arange(0, dim, 2)[: (dim // 2)].float() / dim)),
        1,
    )

    # ===== 步骤 2：YaRN 外推（如果启用） =====
    if rope_scaling:
        orig_max, factor, beta_fast, beta_slow, attn_factor = (
            rope_scaling.get("original_max_position_embeddings", 2048),
            rope_scaling.get("factor", 16),  # 扩展倍数 s
            rope_scaling.get("beta_fast", 32.0),  # 高频边界参数 β_fast
            rope_scaling.get("beta_slow", 1.0),  # 低频边界参数 β_slow
            rope_scaling.get("attention_factor", 1.0),  # 温度修正因子
        )

        # 如果目标长度超过训练长度，应用 YaRN 外推
        if end / orig_max > 1.0:
            # 计算分段边界：inv_dim(β) = d·ln(L_train / (β·2π)) / (2·ln(base))
            inv_dim = lambda b: (
                (dim * math.log(orig_max / (b * 2 * math.pi))) / (2 * math.log(rope_base))
            )
            low = max(math.floor(inv_dim(beta_fast)), 0)
            high = min(math.ceil(inv_dim(beta_slow)), dim // 2 - 1)

            # 斜坡函数：γ(j) = clamp((j - low) / (high - low), 0, 1)
            # 在 low 之前，ramp 为 0；在 high 之后，ramp 为 1；在 low 和 high 之间，线性过渡。
            # clamp 函数限制了数值只能在 [0, 1] 之间。
            ramp = torch.clamp(
                (torch.arange(dim // 2, device=freqs.device).float() - low)
                / max(high - low, 0.001),
                0,
                1,
            )

            # 应用 YaRN 分段缩放：θ'_j = θ_j · [(1-γ) + γ/s]
            freqs = freqs * (1 - ramp + ramp / factor)

    # ===== 步骤 3：计算所有位置的频率 =====
    # Φ_{m,j} = m · θ_j（外积生成角度矩阵）
    t = torch.arange(end, device=freqs.device)  # 位置索引 [0, 1, 2, ..., end-1]
    freqs = torch.outer(t, freqs)  # shape = [end, dim//2]

    # ===== 步骤 4：输出 cos(Φ) 和 sin(Φ)，乘以温度修正因子 =====
    # 由于 RoPE 使用复数旋转，需要将 dim/2 的频率复制到完整的 dim 维度
    freqs_cos = torch.cat([freqs.cos(), freqs.cos()], dim=-1) * attn_factor
    freqs_sin = torch.cat([freqs.sin(), freqs.sin()], dim=-1) * attn_factor

    return freqs_cos, freqs_sin


def apply_rotary_pos_emb(q, k, cos, sin, position_ids=None, unsqueeze_dim=1):
    """
    对 Query 和 Key 应用旋转位置编码。

    数学公式：
        RoPE(x, m) = x ⊙ cos(mθ) + rotate_half(x) ⊙ sin(mθ)

    这是复数旋转 x · e^{imθ} 的实数等价形式。

    Args:
        q: Query [batch, seq_len, num_heads, head_dim]
        k: Key   [batch, seq_len, num_kv_heads, head_dim]
        cos: 预计算的 cos 值 [seq_len, head_dim]
        sin: 预计算的 sin 值 [seq_len, head_dim]
        position_ids: 位置索引（未使用，cos/sin 已包含位置信息）
        unsqueeze_dim: 扩展维度以匹配 q/k 形状（默认 1）

    Returns:
        q_embed: 应用 RoPE 后的 Query [batch, seq_len, num_heads, head_dim]
        k_embed: 应用 RoPE 后的 Key [batch, seq_len, num_kv_heads, head_dim]
    """

    def rotate_half(x):
        """
        将向量后半部分取负放到前面：[a,b,c,d] → [-c,-d,a,b]
        这实现了复数旋转的实部/虚部交换。
        """
        return torch.cat((-x[..., x.shape[-1] // 2 :], x[..., : x.shape[-1] // 2]), dim=-1)

    # 调整 cos 和 sin 的形状以匹配 q/k：[seq_len, head_dim] -> [seq_len, 1, head_dim]
    cos = cos.unsqueeze(unsqueeze_dim)
    sin = sin.unsqueeze(unsqueeze_dim)

    # RoPE(x, m) = x ⊙ cos(mθ) + rotate_half(x) ⊙ sin(mθ)
    q_embed = (q * cos) + (rotate_half(q) * sin)
    k_embed = (k * cos) + (rotate_half(k) * sin)

    return q_embed, k_embed
```
