# 一、 理论基础：神经网络与语言模型

## 1. 什么是神经网络？

- 从本质上看，神经网络是一种用于**拟合任意（复杂）函数**的数学模型。
	- **哲学视角**：世界上的许多规律都可以抽象为函数（World is a function）。当我们用神经网络去拟合这些极其复杂的函数时，本质上也就是在让模型“拟合”并理解这个世界的运行规律。

## 2. 语言模型的核心逻辑

- **基本原理**：基于已有的上下文句子，预测下一个最可能出现的词（Next-token prediction），其过程类似于做“完形填空”。

## 3. Attention（注意力机制）是什么？

- Attention 是现代大语言模型（如 Transformer 架构）的灵魂。
	- 核心机制：加权求和。
	- 动态分配：当模型生成当前输出时，它能够打破距离限制，动态地为之前所有的输入分配不同的“注意力权重”（Attention Weights）。
	- 上下文聚合：通过这种机制，模型能够精准地提取并聚合与当前生成任务最相关的上下文信息。

# 二、 PyTorch 基础组件解析

在 PyTorch 中，构建和训练神经网络离不开以下五大核心组件：

## 1. Tensor (张量)

- 定义：PyTorch 中最基础的数据结构，类似于 NumPy 的多维数组，但支持 GPU 加速计算。
- 作用：作为数据的载体，存储输入数据、模型权重以及所有中间计算结果等数值信息。
- 关键点：
	- 可通过 `torch.tensor()` 创建。
	- 支持 `.to('cuda')` 在 CPU 与 GPU 间迁移。
	- 是 `Autograd` 系统追踪的对象。

## 2. Parameter (可优化参数)

- 定义：`Tensor` 的子类，专门用于表示模型中需要学习和更新的权重（Weights）和偏置（Biases）。
- 作用：它是模型“记忆”的核心载体。会自动注册到 `Module` 中，调用 `model.parameters()` 时会被自动提取并传递给优化器。
- 关键点：
	- 通常通过 `nn.Parameter(torch.randn(...))` 创建。
	- 默认 `requires_grad=True`（需要计算梯度）。

## 3. Module (模型模块)

- 定义：`nn.Module` 是所有神经网络模块的基类，可视作管理张量和层的容器。
- 作用：封装网络结构（如卷积层、全连接层）和前向传播逻辑（`forward` 函数）。
- 关键点：
	- 自定义模型时需继承该类，并强制实现前向传播方法 `def forward(self, x):`。
	- 自动管理内部包含的 `Parameter`。
	- **常用方法**：`.train()`（训练模式）、`.eval()`（评估/推理模式）、`.to(device)`（设备迁移）、`.state_dict()`（保存/加载模型权重）。

## 4. Autograd (自动求导)

- 定义：PyTorch 的自动微分引擎，用于动态构建计算图并计算梯度。
- 作用：在反向传播阶段，自动计算损失函数（Loss）相对于每个 `Parameter` 的梯度（偏导数）。
- 关键点：
	- 核心机制：在后台默默追踪所有对 `requires_grad=True` 的张量进行的操作。
	- 触发方式：通过调用 `loss.backward()` 触发计算。
	- 结果存储：梯度计算后存储在张量的 `.grad` 属性中。

## 5. Optimizer (优化器)

- 定义：参数更新算法（如 SGD, Adam, RMSprop 等）的工具类集合。
- 作用：根据 `Autograd` 计算出的梯度，更新模型中的 `Parameter` 以最小化损失。
- 关键点：
	- 位于 `torch.optim` 包中（如 `optim.Adam`）。
	- 初始化时必须绑定模型的参数，例如：`optim.Adam(model.parameters(), lr=0.001)`。

## 三、核心工作流：它们是如何协作的？

为了更直观地理解，以下是这五个组件在标准训练循环（Training Loop）中的协同顺序：
1. 数据准备：将原始数据转换为 Tensor，并送入目标设备（如 GPU）。
2. 前向传播 (Forward)：数据流经 Module，利用内部的 Parameter 执行 `forward` 函数中的计算逻辑，输出预测值。
3. 计算损失 (Compute Loss)：对比预测值与真实标签，计算出误差 Loss（它本身也是一个包含计算图历史的 Tensor）。
4. 反向传播 (Backward)：调用 `loss.backward()` 触发 Autograd 引擎，逆向计算 Loss 对所有参与计算的 Parameter 的梯度。
5. 参数更新 (Optimize)：
    - Optimizer 执行 `step()`，读取 `.grad` 中的梯度并更新 Parameter 的值。
    - 执行 `optimizer.zero_grad()` 清空本轮梯度，防止与下一轮迭代冲突（此步骤通常在反向传播前或更新后进行）。
