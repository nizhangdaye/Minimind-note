# 一、为什么需要归一化？

- **归一化**：将数据调整为均值为 0、标准差为 1 的标准分布，使散乱的数据分布更加规整，有利于模型训练。
- 深度神经网络层数增多后，信号在逐层传播中容易产生非线性失真，导致梯度在反向传播时发生**爆炸**或**消失**：
	- $$\frac{dL}{dw} = \frac{dL}{dy} \times \frac{dy}{dw} = \frac{dL}{dy} \cdot x$$
	- 梯度与输入 $x$ 的数值大小直接相关——若 $x$ 过大则梯度爆炸，$x$ 过小则梯度消失。
	- 因此需要归一化来约束 $x$ 的范围，使标准差趋近于 1，从而稳定梯度。

# 二、RMSNorm

RMSNorm（Root Mean Square Normalization）相较于传统的 LayerNorm，省去了均值的计算步骤，实验表明效果相当。

$$\text{RMS}(\mathbf{x}) = \sqrt{\frac{1}{d} \sum_{i=1}^{d} x_i^2 + \epsilon}$$

$$\bar{x}_i = \frac{x_i}{\text{RMS}(\mathbf{x})} \cdot \gamma_i$$

```python
class RMSNorm(nn.Module):
    def __init__(self, dim, eps: float = 1e-5):
        """
        Args:
            dim: 输入张量的最后一个维度大小
            eps: 防止除零的数值稳定性常数
        """
        super().__init__()
        self.eps = eps

        # 可学习的缩放参数，初始化为全 1
        self.weight = nn.Parameter(torch.ones(dim))

    def _norm(self, x):
        """
        公式: y = x / sqrt(mean(x^2) + eps)
        Args:
            x: 输入张量，形状为 (batch_size, seq_length, dim)
        """
        # x.pow(2)：计算 x 的平方
        # .mean(dim=-1, keepdim=True)：在最后一个维度上计算均值，保持维度不变
        # torch.rsqrt()：计算平方根的倒数，即 1/sqrt(x)
        # 直接调用 rsqrt 比先 sqrt 再取倒数更高效，尤其在 GPU 上
        return x * torch.rsqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)

    def forward(self, x):
        """
        Args:
            x: 输入张量，形状为 (batch_size, seq_length, dim)
        """
        # 先转为 float32 进行归一化计算（提高数值稳定性）
        # 然后转回原始精度
        return self.weight * self._norm(x.float()).type_as(x)
```

# 三、RMSNorm 与 LayerNorm 的异同

## LayerNorm 公式与实现

$$\text{LayerNorm}(\mathbf{x}) = \gamma \left( \frac{\mathbf{x} - \mu}{\sqrt{\sigma^2 + \epsilon}} \right) + \beta$$

```python
class LayerNorm(nn.Module):
    def __init__(self, dim, eps: float = 1e-5):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))
        self.bias = nn.Parameter(torch.zeros(dim))

    def forward(self, x):
        """
        公式: y = (x - mean) / sqrt(var + eps) * weight + bias
        Args:
            x: 输入张量，形状为 (batch_size, seq_length, dim)
        """
        mean = x.mean(dim=-1, keepdim=True)  # 均值
        var = x.var(dim=-1, keepdim=True, unbiased=False)  # 方差
        normed = (x - mean) / torch.sqrt(var + self.eps)
        return self.weight * normed + self.bias
```

## 相同点

RMSNorm 和 LayerNorm 的归一化操作都是沿着每个 token 内部的**特征维度**（即 `embedding_dim`）进行的，而非沿 batch 维度。它们对每个 token 独立进行标准化。

## 不同点

相较于 LayerNorm，RMSNorm **不做减均值操作**。

> [!tip] 直觉理解
> 观察公式可以发现，若将 LayerNorm 中的均值 $\mu$ 设为 0，即退化为 RMSNorm。

# 四、为什么使用 RMSNorm 而不是 LayerNorm？

## 理论依据

RMSNorm 的作者通过大量消融实验发现，LayerNorm 的成功主要归功于**重缩放（Re-scaling）不变性**，而非重中心化（Re-centering）不变性。在深度神经网络中，激活值的均值通常在 0 附近波动，强制中心化的收益微乎其微，但计算均值却引入了额外的规约（Reduction）操作，增加了计算复杂度。

## 效率优势：内存带宽与计算的权衡

从理论 FLOPs 来看，RMSNorm 相对于 LayerNorm 的减少看似微不足道（Transformer 的计算瓶颈主要在于庞大的矩阵乘法）。然而在实际硬件（GPU / TPU）上，RMSNorm 带来了显著的速度提升（==推理速度提升约 10%–40%==）。

根源在于：**LayerNorm 是内存带宽受限（Memory-Bandwidth Bound）的操作**——

- LayerNorm 需要**两次**全向量扫描：一次计算均值 $\mu$，一次计算方差 $\sigma^2$（依赖于 $\mu$），需频繁从显存读取数据并同步。
- RMSNorm 只需**一次**扫描计算平方和，减少了数据读写次数和同步屏障（Synchronization Barriers），显著降低了内存访问开销。

> [!note] 工程优化加成
> RMSNorm 的简洁性使其更容易进行**算子融合（Kernel Fusion）**。在 FlashAttention、vLLM 等高性能推理框架中，RMSNorm 通常被深度优化，进一步拉大了与 LayerNorm 的性能差距。

# 五、为什么大语言模型中不使用 BatchNorm？

## BatchNorm 在计算机视觉（CV）中的应用

在计算机视觉领域，Batch Normalization (BN) 曾长期占据统治地位。BN 沿着**批次维度**对特征进行标准化，利用当前 batch 的统计量来近似全局分布。

- **应对空间相关性**：图像数据具有强烈的空间相关性（相邻像素值或模式相似），像素特征在一个 batch 中分布相似，因此沿 batch 维度归一化非常合理。
- **加速模型收敛**：BN 通过计算每个特征（如每个通道）的均值和方差，有效减轻了空间相关性的影响，保证了每层输入分布的稳定性。

## BatchNorm 在自然语言处理（NLP）中的局限性

在 NLP 任务和 Transformer 架构中，BN 显得"水土不服"。NLP 中的每个 token 都是包含特定语义和上下文信息的独立单位，不同 token 语义各异，本应独立归一化。使用 BN 会引发以下问题：

- **变长序列的 Padding 干扰**：文本长度不固定，batch 训练时短序列需通过 Padding 补齐。BN 跨 batch 计算时，无意义的 Padding token 会扭曲均值和方差。即便使用 Masking 技术缓解，也会导致实现复杂且效率低下。
- **序列维度的独立性被破坏**：token 的语义依赖于自身上下文，不同样本间的 token 并无对应关系（如样本 A 的第 5 个词与样本 B 的第 5 个词可能完全无关）。BN 强制跨 batch 统计，会破坏 token 间的独立性。
- **小 Batch Size 的统计不稳定性**：LLM 训练常受限于显存容量，Micro-Batch Size 极小（甚至为 1）。此时 BN 估算的统计量噪声极大，无法代表真实数据分布，易导致训练震荡甚至发散。

## Layer Normalization 的优势

为解决上述问题，Layer Normalization 应运而生，并在 NLP 领域取代了 BN：

- **按特征维度归一化**：LN 沿特征维度对单个样本进行归一化。
- **独立计算**：统计量仅依赖当前样本自身，完全独立于 batch size 和序列长度。
- **天然契合 NLP**：完美避开了 Padding 干扰和 batch size 限制，成为构建 Transformer 架构的天然选择。
